------------------------------------------------------------------------------

# Purpose

The Design System ensures every screen in HadionERP behaves, feels,
and communicates like one enterprise platform.

The Design System is not a UI kit.

It is the language of the product.

------------------------------------------------------------------------------

# Design Philosophy

Enterprise software should reduce thinking.

Users should recognize patterns immediately.

Users should never ask:

"Where is Save?"

"Where are attachments?"

"How do I approve?"

The answer should always be obvious because every screen follows the
same language.

------------------------------------------------------------------------------

# Core Principles

Consistency before Creativity

Density before Empty Space

Business before Graphics

Information before Animation

Speed before Effects
